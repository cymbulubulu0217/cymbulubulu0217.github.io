走进react

什么是react

React 是一个用于构建用户界面的 JavaScript 库。react官方中文文档

react的特点

1、声明式

写UI像写普通的HTML一样简单方便,提高开发效率

2、组件化

组件可以通过像搭积木一样拼成一个完整的页面

3、一次学习、跨平台编写

只需要一种react语法，就可以通过编译器发布到不同的平台，很大程度上减少了开发成本，并且提高了开发效率

搭建react脚手架

第一步：全局安装
$ npm i -g create-react-app
第二步：创建脚手架
$ create-react-app hello-react
或者
$ npx create-react-app demo-name

区别：npx 会帮助我们临时创建 create-react-app 包，然后项目初始化后会自动删除

第三步：启动项目
$ npm start
或者
$ yarn start

脚手架项目结构

public —- 静态资源文件夹
favicon.icon —— 网站页签图标
index.html ——– 主页面
logo192.png ——- logo图
logo512.png ——- logo图
manifest.json —– 应用加壳的配置文件
robots.txt ——– 爬虫协议文件
src —- 源码文件夹
App.css ——– App组件的样式
App.js ——— App组件
App.test.js —- 用于给App做测试
index.css —— 样式
index.js ——- 入口文件
logo.svg ——- logo图
reportWebVitals.js
— 页面性能分析文件(需要web-vitals库的支持)
setupTests.js
—- 组件单元测试的文件(需要jest-dom库的支持)

react高效的原因

虚拟DOM

使用虚拟DOM而不是直接操作页面的真实DOM
1、VDOM本质是一个js对象
2、VDOM身上的属性较少，RDOM身上的属性多，因为VDOM是React内部在用，无需RDOM身上的太多属性
3、VDOM最终会被转化为RDOM呈现在页面上

diff算法

diff算法就是更高效的通过对比新旧虚拟DOM来找出真正DOM的变化之处
新的虚拟DOM和旧的虚拟DOM比较时，当两者有相同的虚拟DOM时，会引用旧的真实DOM,再将不同的虚拟DOM转化为真实DOM再追加到页面上

babel

用途：用于将jsx转化为js

jsx

react为什么不用js而是用jsx？
创建VDOM：
jsx(javaScript XML):
const VDOM = <h1>hello jsx</h1>
js:
const VDOM = React.create('h1',{id:"title"},'hello js')
两者写法对比我们可以发现js的写法过于复杂且不直观

jsx会通过内置的babel被转化为React.createElement() 的函数并调用，返回了VDOM，再通过 ReactDOM.render() 的方法转化为RDOM并渲染到页面上

原因：因为js创建VDOM太繁琐了，有了jsx可以更简单方便直观的创建VDOM(本质是 React.createElement() 的语法糖)
17版本前是通过手动引入Babel的形式
17版本后不需要手动引入，因为react脚手架内置了@babel/plugin-transform-react-jsx插件
我们可以在babel的网站，在线测试babel.js,这个网站可以把 JSX 代码转换成 JS 代码

jsx的语法规则

1、定义虚拟DOM，不要写引号
2、标签中混入js表达式时要用 {}。
3、样式的类名不能用 class，要用 className。

  • class ---> className,for ---> htmlFor 特殊属性

4、内联样式要有 style={{key:value}}的样式去写。
5、只有一个根标签,可以用<><>来包裹，也可以用<React.Fragment></React.Fragment>
6、标签必须闭合
7、标签首字母小写为html标签，大写为组件标签
8、如果jsx有换行，最好使用 () 包裹

jsx 嵌入表达式

JSX条件渲染

  • if/else 条件渲染
  • 三元运算符条件渲染
  • 逻辑运算符完成渲染

    JSX列表渲染

    使用 map 渲染列表 总结： 在JSX中一般使用 map 来遍历进行渲染，遍历的时候需要给子元素加上 key={唯一标识}，提升diff性

JSX样式-动态className

1、在使用className的时候会遇见多个类名动态绑定，可以模仿vue使用对象的方式：{类名：布尔值}
2、我们也可以使用 classnames库来完成属性的动态绑定

  • 安装classnames文档 认识 classnames API 总结： 推荐以后开发项目中使用第三方的库来解决className的值绑定问题
